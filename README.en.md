## 🧮 Voxel World + Perlin Noise

### Voxel World

3D space divided into **unit-cube grid**, each cell assigned a block type:

```
world[x][y][z] = BLOCK_TYPE  // 0 = air, 1 = grass, 2 = dirt, 3 = stone, 4 = sand
```

- **x, z** (horizontal) — `-WORLD_SIZE/2` ~ `+WORLD_SIZE/2`
- **y** (vertical) — `0` (ground) ~ `MAX_HEIGHT` (mountain peak)
- Each cell is 1×1×1 unit cube

### Perlin Noise (Terrain Auto-Generation)

Ken Perlin's **gradient noise** (1983) for natural mountains·lakes:

```
height(x, z) = noise(x * FREQ, z * FREQ) * AMP
              + noise(x * FREQ * 2, z * FREQ * 2) * AMP * 0.5   // octave 2
              + noise(x * FREQ * 4, z * FREQ * 4) * AMP * 0.25  // octave 3
```

**fBm (fractional Brownian motion)** — multiple octaves add detail:

```js
function generateHeight(x, z) {
  let height = 0;
  let amplitude = 1;
  let frequency = 0.02;

  for (let octave = 0; octave < 4; octave++) {
    height += noise(x * frequency, z * frequency) * amplitude;
    amplitude *= 0.5;     // amplitude halves (detail weakens)
    frequency *= 2;       // frequency doubles (more detail)
  }
  return Math.floor(height * 10);
}

// Block placement
for (let x = -32; x < 32; x++) {
  for (let z = -32; z < 32; z++) {
    const h = generateHeight(x, z);
    for (let y = 0; y <= h; y++) {
      let type = (y === h) ? GRASS :      // top = grass
                (y > h - 3) ? DIRT :       // 2 below = dirt
                STONE;                      // rest = stone
      world[x + 32][y][z + 32] = type;
    }
  }
}
```

## ⚡ Performance Optimization

Thousands of blocks at 60fps requires **mesh optimization**:

### ❌ Bad: Individual Meshes

```js
for (const block of blocks) {
  const mesh = new THREE.Mesh(geometry, material);
  scene.add(mesh);  // 5000 draw calls → 5fps
}
```

### ✅ Good 1: InstancedMesh

`InstancedMesh` renders **multiple instances of same geometry/material in a single draw call**:

```js
const geom = new THREE.BoxGeometry(1, 1, 1);
const mat = new THREE.MeshLambertMaterial();
const instancedMesh = new THREE.InstancedMesh(geom, mat, MAX_BLOCKS);

const dummy = new THREE.Object3D();
blocks.forEach((block, i) => {
  dummy.position.set(block.x, block.y, block.z);
  dummy.updateMatrix();
  instancedMesh.setMatrixAt(i, dummy.matrix);
});
instancedMesh.instanceMatrix.needsUpdate = true;
scene.add(instancedMesh);  // 1 draw call → 60fps
```

### ✅ Good 2: Merge Geometries

```js
import { BufferGeometryUtils } from 'three/addons/utils/BufferGeometryUtils.js';
const merged = BufferGeometryUtils.mergeGeometries(blockGeometries);
const mesh = new THREE.Mesh(merged, material);  // 1 mesh
```

### 🎯 Raycaster (Block Face Detection)

`Raycaster` casts a ray from camera through cursor, returning **exactly which block and which face** was hit:

```js
const raycaster = new THREE.Raycaster();
const mouse = new THREE.Vector2();

canvas.addEventListener('mousedown', (e) => {
  mouse.x = (e.clientX / window.innerWidth) * 2 - 1;
  mouse.y = -(e.clientY / window.innerHeight) * 2 + 1;
  raycaster.setFromCamera(mouse, camera);

  const intersects = raycaster.intersectObject(instancedMesh);
  if (intersects.length > 0) {
    const hit = intersects[0];
    const blockPos = hit.object.userData.blocks[hit.instanceId];
    const faceNormal = hit.face.normal;  // which face?

    if (e.button === 0) removeBlock(blockPos);
    else if (e.button === 2) addBlock(blockPos, faceNormal);
  }
});
```

## 🎨 Design Choices

Four decisions made during brainstorming:

| Decision | Choice | Rationale |
|---|---|---|
| **Rendering API** | Three.js + InstancedMesh (vs individual Mesh / MergeGeometry) | Prompt requirement. InstancedMesh is simplest and fastest (1 draw call) |
| **Terrain gen** | Perlin Noise (vs Simplex / Value / Worley) | Prompt requirement. fBm 4 octaves naturally creates mountains·lakes |
| **Interaction** | Raycaster (vs bounding-box intersection) | Face normal vector tells exactly which face → required for placement position |
| **Storage** | In-memory 3D array (vs serialization) | Single-session sandbox, no persistent save needed, memory is fastest |

### Customization

```js
const CONFIG = {
  WORLD_SIZE: 64,             // Horizontal size (cells, 64 = 4096 blocks/layer)
  MAX_HEIGHT: 32,             // Max vertical height (cells)
  NOISE_FREQ: 0.02,           // Perlin noise frequency (smaller = larger mountains)
  NOISE_AMP: 10,              // Perlin amplitude (larger = taller mountains)
  NOISE_OCTAVES: 4,           // fBm octave count
  WATER_LEVEL: 8,             // Lake surface height
  BLOCK_SIZE: 1,              // Block side length
  MAX_BLOCKS: 50000,          // InstancedMesh max instances
  GRAVITY: 0.05,              // Jump gravity (optional)
  MOVE_SPEED: 0.1,            // Move speed
  MOUSE_SENSITIVITY: 0.002,   // Mouse sensitivity
};
```

**Parameter exploration**:
- `WORLD_SIZE = 32` → small island (perf headroom)
- `WORLD_SIZE = 64` → mid map (default, ~4K blocks/layer)
- `WORLD_SIZE = 128` → large world (~16K blocks/layer, 60fps limit)
- `NOISE_FREQ = 0.01` → flat terrain (gentle hills)
- `NOISE_FREQ = 0.05` → rough mountains (sharp peaks)
- `NOISE_OCTAVES = 1` → smooth landscape
- `NOISE_OCTAVES = 8` → very detailed (compute cost ↑)
- `WATER_LEVEL = 4` → shallow lake
- `WATER_LEVEL = 16` → deep lake + underwater mountains

**Performance comparison (64×64×32 blocks)**:

| Method | Draw calls | FPS |
|---|---|---|
| Individual Mesh × 130K | 130,000 | <5 fps ❌ |
| MergeGeometry | 1 | 60 fps ✅ |
| InstancedMesh | 1 | 60 fps ✅ |
| Chunked (16×16×32) | ~256 | 60 fps ✅ (huge worlds) |

## 📚 Related Techniques

| Technique | Purpose | Complexity |
|---|---|---|
| **Perlin Noise** | Natural terrain (mountains·valleys) | O(N) |
| **Simplex Noise** | Perlin improved (faster) | O(N) |
| **Value Noise** | Simplest noise | O(N) |
| **fBm** | Multiple octaves (detail accumulation) | O(N × octaves) |
| **Marching Cubes** | Smooth mesh (Minecraft variant) | O(N³) |
| **Greedy Meshing** | Adjacent block merge (perf) | O(N) |
| **Chunk System** | Large world management (16×16) | O(chunks) |

**Perlin vs Simplex**:
- **Perlin (1983)**: Ken Perlin, gradient interpolation, smooth natural landscapes
- **Simplex (2001)**: Ken Perlin improved, faster with higher-dim support

**Voxel Rendering evolution**:
- Minecraft (2011) — simple chunks + adjacent face culling
- Minecraft RTX (2020) — Path tracing + DXR
- Teardown (2022) — fully destructible voxels
